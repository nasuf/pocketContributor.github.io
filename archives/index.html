Oracle User Management

Oct 23, 2016
创建用户

create user c##mldnuser

identified by java_android

default tablespace mldn_data

temporary tablespace mldn_temp

quota 30M on mldn_data

quota 20M on users

account unlock

password expire;

配置说明：

default tablespace mldn_data: 该用户默认使用的表空间；如果用户在创建数据库对象时没有指定表空间，则默认使用mldn_data表空间存储；

tempporary tablespace mldn_temp: 该用户使用order by 或者group by 子句存储量过大时保存的临时表空间；

quota 30M on mldn_data: 该用户在mldn_data表空间上最多使用30MB大小；

quota 20M on users: 该用户在users表空间上最多使用20MB大小；

account unlock: 该用户默认为活动账户；

password expire: 用户登录后需要强制用户修改密码；

查看用户信息

select username,user_id,default_tablespace,temporary_tablespace,created,lock_date,profile from dba_users where username='c##mldnuser';

修改用户密码

alter user c##mldnuser identified by hellojava;

控制用户锁定

alter user c##mldnuser account lock;

让用户密码失效

alter user c##mldnuser password expire;

删除用户

drop user c##mldnuser;

为用户授权

grant create session to c##mldnuser;

grant create table, create sequence, create view to c##mldnuser with admin option; #with admin option子句表示该用户可以将其权限授予其他用户；

查询用户权限

select * from dba_sys_privs where grantee in ('c##mldnuser') order by grantee desc;

撤销用户权限

revoke create table, create view from c##mldnuser;

数据类型
NUMBER

NUMBER可以定义整形NUMBER(n), 也可以定义浮点型NUMBER(n,m)，后者表示一共有n位，其中m位小数；

BINARY_INTEGER & PLS_INTEGER

两者采用二进制的补码形式存储，而NUMBER采用十进制进行存储，使用时先转为二进制再进行计算，所以前者比NUMBER性能更高；

BINARY_FLOAT & BINARY_DOUBLE

Oracle 10g后引入，比NUMBER节约空间，表示的范围更大，直接采用二进制方式存储；声明方式：

v_float BINARY_FLOAT := 8909.82F;

v_double BINARY_DOUBLE := 92884.23D;

CHAR & VARCHAR2

CHAR类型使用定长方式保存字符串，如果用户为其设置的内容不足其定义长度，则会自动补充空格；VARCHAR2是变长字符串，如果为其设置的内容不足其长度，不会为其补充内容；

NCHAR & NVARCHAR2

两者操作特点与CHAR和VARCHAR2一样，只是两者保存的数据为UNICODE编码，即中文或英文都会变为十六进制编码保存。使用UNICODE编码的方便之处是统一了字符与汉字的长度，每一位中文都只占一位；但同时，由于所有的字母都会按照十六进制保存，所以在某种程度上讲也会浪费空间；
